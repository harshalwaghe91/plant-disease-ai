## Project Overview

This system uses a **MobileNetV2 CNN** (Transfer Learning) to classify plant leaf images into 8 categories:

| Class | Plant | Type |
|---|---|---|
| Healthy | General | Healthy |
| Tomato_Early_Blight | Tomato | Disease |
| Tomato_Late_Blight | Tomato | Disease |
| Tomato_Leaf_Mold | Tomato | Disease |
| Potato_Early_Blight | Potato | Disease |
| Potato_Late_Blight | Potato | Disease |
| Pepper_Bacterial_Spot | Pepper | Disease |
| Pepper_Healthy | Pepper | Healthy |

**Target Accuracy: >95%**

## Dataset Preparation

### Download PlantVillage Dataset

1. Go to: https://www.kaggle.com/datasets/emmarex/plantdisease
2. Download and extract to `dataset/` folder
3. Organize into `train/` and `val/` subfolders (80/20 split)

### Folder Structure Required

```
dataset/
├── train/
│   ├── Healthy/
│   │   ├── img001.jpg
│   │   └── ...
│   ├── Tomato_Early_Blight/
│   │   └── ...
│   ├── Tomato_Late_Blight/
│   ├── Tomato_Leaf_Mold/
│   ├── Potato_Early_Blight/
│   ├── Potato_Late_Blight/
│   ├── Pepper_Bacterial_Spot/
│   └── Pepper_Healthy/
│
└── val/
    ├── Healthy/
    ├── Tomato_Early_Blight/
    └── ... (same structure)
```

### Quick Split Script

```python
# run_once: split.py
import os, shutil, random
SRC = 'dataset/raw'   # Your raw PlantVillage folder
for cls in os.listdir(SRC):
    imgs = os.listdir(os.path.join(SRC, cls))
    random.shuffle(imgs)
    split = int(len(imgs) * 0.8)
    for split_name, files in [('train', imgs[:split]), ('val', imgs[split:])]:
        dest = f'dataset/{split_name}/{cls}'
        os.makedirs(dest, exist_ok=True)
        for f in files:
            shutil.copy(os.path.join(SRC, cls, f), os.path.join(dest, f))
print("Done!")
```

## Model Training

```bash
# Basic training (30 epochs, batch size 32)
python train.py

# Custom parameters
python train.py --data_dir dataset --epochs 50 --batch_size 16 --img_size 224

# All options
python train.py --help
```

### Training Output

| File | Description |
|---|---|
| `model/model.h5` | Final trained model |
| `model/class_names.json` | Class order for inference |
| `model/training_curves.png` | Accuracy & loss graphs |
| `model/confusion_matrix.png` | Per-class confusion matrix |
| `model/training_log.csv` | Epoch-by-epoch metrics |
| `model/training_summary.json` | Final accuracy summary |

### Training Strategy (Two-Phase)

**Phase 1** (Epochs 1–15): Only the classification head is trained. MobileNetV2 base is frozen.

**Phase 2** (Epochs 16–30): Fine-tuning — last 30 layers of MobileNetV2 are unfrozen with 10× lower learning rate.

Expected results with PlantVillage:
- Validation Accuracy: **96–98%**
- Training Time: ~45 min on GPU, ~3–4 hours on CPU

## Deployment on Render

### Step 1: Prepare Repository

```bash
# Initialize git
git init
git add .
git commit -m "Initial commit: Plant Disease AI"

# Create GitHub repo and push
git remote add origin https://github.com/yourusername/plant-disease-ai.git
git push -u origin main
```

> **Important:** Add `model/model.h5` using [Git LFS](https://git-lfs.com/) or upload manually to Render after deployment.

### Step 2: Deploy on Render

1. Go to https://render.com → Sign up / Login
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub account
4. Select your repository
5. Configure:
   - **Name**: `plant-disease-ai`
   - **Environment**: `Python`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120`
   - **Plan**: Starter (free)

### Step 3: Set Environment Variables

In Render Dashboard → Your Service → **Environment**:

| Key | Value |
|---|---|
| `SECRET_KEY` | (click "Generate") |
| `ADMIN_USERNAME` | `admin` |
| `ADMIN_PASSWORD` | `yourStrongPassword` |
| `FLASK_DEBUG` | `false` |

### Step 4: Upload Model File

Option A — Render Disk:
- Add a disk in Render → Mount path: `/opt/render/project/src/model`
- Upload `model.h5` via Render Shell

Option B — Store in cloud (Google Drive / S3) and download at startup:
- Add startup download in `app.py` `load_model()` function

### Step 5: Access App

Your app will be live at: `https://plant-disease-ai.onrender.com`

## Adding New Diseases

To add a new disease **without changing any other file**:

1. Open `disease_data.py`
2. Add an entry to `DISEASE_DATABASE`:

```python
"Tomato_Spider_Mite": {
    "display_name": "Tomato Spider Mite",
    "plant": "Tomato",
    "description": "Spider mites are tiny arachnids...",
    "symptoms": ["Tiny yellow dots on leaves", "Fine webbing under leaves"],
    "causes": ["Tetranychus urticae mite", "Hot, dry conditions"],
    "severity": "Moderate",
    "chemical_treatment": "Abamectin or Bifenazate miticide...",
    "organic_treatment": "Neem oil spray, insecticidal soap...",
    "prevention": ["Maintain humidity", "Inspect plants regularly"],
    "color": "#e11d48"
}
```

3. Add images to `dataset/train/Tomato_Spider_Mite/` and `dataset/val/Tomato_Spider_Mite/`
4. Retrain the model with `python train.py`

## Troubleshooting

### Model not loading
```
[WARNING] model.h5 not found
```
**Fix:** Run `python train.py` first. App works in demo mode without model.

### TensorFlow installation issues
```bash
# For CPU-only (no GPU) — faster install
pip install tensorflow-cpu==2.15.0
```

### Port already in use
```bash
# Kill existing process
lsof -ti:5000 | xargs kill -9
```

### PDF download fails
```bash
pip install reportlab
```

### Render deployment — slug size too large
TensorFlow is large (~500MB). Use Render's **Starter** plan which supports up to 2GB.
Alternatively, use `tensorflow-cpu` in `requirements.txt`.

### Out of memory during training
Reduce batch size:
```bash
python train.py --batch_size 16
```
